1. 알고리즘 문제를 효율적으로 푸는 방법


  • 자료구조, 알고리즘에 대한 지식이 있을 때 도전하기
  • 한 문제를 2시간 이상 붙잡고 있지 말기
  • 모르는 문제의 해설을 확인한 뒤에는 스스로의 힘으로 다시 풀어보기
  • 비슷한 유형의 문제를 여러 개 풀면서 다양한 유형에 익숙해지기

알고리즘 문제를 푸는 것은 수학문제를 푸는 것과 크게 다르지 않습니다. 기본기가 부족한 상태에서는 문제를 풀면서도 이게 무슨 어떤 문제이고 어떤 점에서 좋은 문제인지 알지 못하며, 문제를 풀더라도 조금만 변형된 유형이 나오면 무너지고 맙니다.

가장 먼저, 기본 배경지식이 없다면 섣불리 도전하지 말아야 합니다. 알고리즘 문제를 풀기 위해서는 알고리즘을 공부해야 하고, 알고리즘을 공부하기 위해서는 자료구조를 공부해야 하고 자료구조를 공부하기 위해서는 프로그래밍 언어를 1개 이상 알아야 합니다. 선수 과목에 대한 지식이 없이는 학업 능률의 효율적 향상을 기대하기 어렵습니다.

한 문제를 2시간 이상 잡고 있는 것 또한 비효율적입니다. 알고리즘 문제를 푸는데에는 시간이 많이 걸리지 때문에, A형 수준의 문제를 2시간 이상 풀고 있는 것은 시간을 비효율적으로 사용하는 것입니다. 빠르게 포기하고, 좋은 풀이를 참고하는 것이 효율적으로 실력을 쌓을 수 있는 길입니다.

모르는 문제의 풀이를 찾아본 후에는 꼭!! 스스로의 힘으로 다시 풀어봐야 합니다. 풀이를 복습할 수 있을 뿐만 아니라 구현력을 키우는 아주 좋은 방법입니다.

비슷한 유형의 문제를 많이 풀어보는 것 또한 중요합니다. 알고리즘을 사용할 수 있는 다양한 예시를 익히는 것을 통해 심화 알고리즘을 학습하고, 시간, 공간적으로 우수한 풀이를 만들어나갈 수 있는 지름길이기 때문입니다.


2. C++로 문제를 풀 때에 유용한 헤더

사람들이 알고리즘 문제를 풀 때 가장 많이 사용하는 언어는 C++입니다. 속도가 빠르고 다양한 헤더파일을 사용하여 복잡한 연산도 간결하게 표현할 수 있기 때문인데요, 알고리즘 문제 풀이에서 가장 많이 쓰이는 헤더파일과 간단한 설명, 그리고 포함된 기능을 몇가지 소개하고자 합니다. 직접 사용해보세요!

  1. algorithms - 기본 정렬 및 비교, 상/하한 선언 - sort, max, min, upper_bound, lower_bound
  2. stack - 자료구조 스택 - push, pop, top
  3. vector - 스택의 확장판, top이 아닌 요소에 접근이 가능하다 - push_back, erase, insert
  4. queue - 자료구조 큐 - 우선순위 큐를 통해 러닝 타임을 절약할 수 있다
  5. maps - 자료구조 딕셔너리 - 자료의 키값과 밸류값이 필요한 문제에서 유용하게 쓰인다
  6. set - 자료구조 이진트리 - 중복확인을 하는데 유용하게 쓰인다
