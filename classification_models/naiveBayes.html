1. 나이브 베이즈(Naive Bayes)란?


'나이브 베이즈(Naive Bayes)'는 확률을 이용한 어림짐작입니다. 왜 확률에 기반하는데, 어림짐작일까요?

나이브 베이즈는 베이지안 정리에 기초하고 있습니다. 베이즈 정리는 이전의 결과를 기반으로 한 현재의 확률을 추정할 때 많이 쓰입니다. 수식으로 나타내면 다음과 같습니다.


2. 베이즈 정리 경험하기

베이즈 정리는 특정 사건이 발생했을 때 이 사건이 어떤 조건 안에서 발생했을 확률이 얼마나 되는지를 계산할 때 유용하게 쓰이는 정리이죠. 예제를 하나 들어보겠습니다. 반도체를 만드는 A공장, B공장이 있다고 하겠습니다. A공장에서는 100개 중에 1개의 불량품이 만들어지고, B공장에서는 100개 중 2개의 불량품이 만들어집니다. A공장과 B공장에서 나온 제품들 중에 무작 위로 하나의 부품을 골랐을 때, 그 부품이 불량품이라면 이 불량품이 A공장에서 나온 불량품일 확률은 얼마나 될까요?

P(A공장의 불량품 | 불량품) = P(A공장의 불량품) / (P(A공장의 불량품) + P(B공장의 불량품)) =1/100 / (1/100+2/100) = 1/3입니다.

한 문제를 더 풀어보겠습니다. 율전동의 날씨는 확률에 따라 변합니다. 오늘의 날씨가 맑을 경우 내일의 날씨가 맑을 확률은 60%, 흐릴 확률은 40%이고, 오늘의 날씨가 흐릴 경우, 내일의 날씨가 맑을 확률은 30%, 흐릴 확률은 70%입니다. 오늘의 날씨가 흐릴 때, 화요일의 날씨가 흐릴 확률은 얼마나 될까요?

P(오늘의 날씨 흐림 | 화요일의 날씨가 흐림) = (P(흐림 -> 맑음 -> 흐림) + P(흐림 -> 흐림 -> 흐림)) /(P(흐림 -> 흐림 -> 흐림) + P(흐림 -> 흐림 -> 맑음) + P(흐림 -> 맑음 -> 흐림) + P(흐림 -> 맑음 -> 맑음)) = (3/10 * 4/10 + 7/10*7/10)/ (7/10*7/10 + 7/10*3/10 + 3/10*4/30 + 3/10*6/10) = 33/100입니다

첫 번째 문제와 두 번째 문제의 차이는 무엇일까요? 첫 번째 문제가 고려해야 할 조건이 독립적이라면, 두 번째 문제는 고려해야 할 조건이 종속적입니다. 조건이 독립적일 경우에는 구해야 할 확률이 다른 사건에 영향을 받지 않으므로, 보다 간편하게 구할 수 있지만, 조건이 독립적인 경우에는 구해야 할 확률이 이전 조건, 혹은 다른 조건의 영향을 받기 때문에 모든 경우를 고려해 주어야 합니다. 계산 식에서도 이런 점이 드러나는데요, 이전 문제가 단순하게 2개의 확률을 구했다면, 두 번째 문제는 4가지의 확률을 모두 구해야 했습니다. 즉, 2의 n승으로 구해야 할 확률이 늘어난다는 이야기입니다.

구해야 할 경우의 수가 2의 n승, 즉 지수스케일로 커진다는 것은 기계학습에 있어서 큰 문제입니다. 실생활에 쓰이는 사례에 영향을 주는 변수는 50개가 될 수도 있고, 100개가 될 수도 있는데, 2의 50승만 해도, 현재의 컴퓨터는 1초에 2의 30승을 계산할 수 있기 때문에 1가지의 경우의 수를 구하기 위해 100만초가 걸리게 됩니다. 계산해야 되는 데이터가 10만개일 경우, 100년이 지나도 계산을 못하게 됩니다.


3. 나이브 베이즈를 사용하는 이유

나이브 베이즈 모델은 모든 변수를 독립적이라고 가정합니다. 엄밀히 따지면 종속적일 수도 있지만, 편의를 위해 독립적이라고 하는것이지요. 이렇게 해야 지수스케일의 연산을 피할 수 있고, 기계학습을 할 수 있기 때문입니다. 변수들 간의 관계를 무시하기 때문에 정확도에서는 손해를 보지만, 학습 속도는 크게 향상되는 것입니다. 그렇기 때문에 Naïve, 즉 어설픈, 순진한이라는 수식어가 앞에 붙고, 설명 또한 어림짐작이라고 할 수 있는 것입니다.
